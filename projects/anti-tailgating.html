Cybersecurity · Figma · Splunk

AI Anti-Tailgating
System

A cybersecurity system design project for reducing physical access control risks. The concept combines AI-assisted entry monitoring, operator-facing UI design, simulated security logs, and Splunk dashboards for incident visibility.

View Demo ↓ Back to Selected Work

Overview

This project explored how secure facilities could detect and respond to tailgating, where an unauthorised person enters behind an authorised user. The solution focused on the full monitoring workflow: access events, visual interface design, alert handling, and dashboard-based analysis.

Key Features

  • Figma interface design for monitoring access points and reviewing alerts.
  • Security workflow design covering detection, alerting, review, and response.
  • Simulated log data for access attempts and potential tailgating incidents.
  • Splunk dashboard views for incident monitoring and trend analysis.
  • Risk-focused design decisions for secure physical environments.

Key Challenges

  • Turning a physical security risk into a clear software monitoring workflow.
  • Designing dashboards that surface incidents without overwhelming operators.
  • Creating realistic log simulation for Splunk analysis and visualisation.
  • Balancing AI-assisted detection concepts with practical response processes.

Demo

The demo presents the designed interface and monitoring concept, including alert review and dashboard-oriented security visibility.

Tech Stack

Figma · Splunk · Cybersecurity Analysis · Log Simulation

My Role

System concept design, Figma UI design, Splunk dashboard preparation, simulated log planning, and security workflow documentation.

Focus Area

Physical access control, security monitoring, incident visibility, dashboard design, and risk reduction workflows.

Outcome

Delivered a complete concept prototype showing how access events can be monitored, analysed, and escalated through operator-facing tools.
